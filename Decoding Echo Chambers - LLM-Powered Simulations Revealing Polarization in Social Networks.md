| 工具 | 用途 |

## 二、各環節細節

### 1. 網路架構

每個個體 $i$ 根據不同架構建構網路：

| 網路架構 | 特性 | 數學式 |
|---|---|---|
| 小世界網路（Small-world） | 高群聚 + 個體間平均距離短 | $L \sim \log N$ |
| 無標度網路（Scale-free） | 冪律分佈，少數個體擁有極多連結 | $P(k) \sim k^{-\gamma}$ |
| 隨機圖形（Random Graph） | 個體根據固定機率隨機與其他個體連接 | 二項式分布 |

### 2. 個體初始化

每個個體 $i$ 被分派獨特的個人檔案（Persona），具備多種 Attributes：

- 姓名、性別、年紀、特質、教育程度
- 根據 **Big Five Personality Model** 設計
- 為消除不同姓名導致的偏見，每個個體只能透過**個體索引**來識別

### 3. 推薦演算法

個體 $i$ 只會和與自身意見相似的鄰居互動：

$$R = \{j \in N(i) \mid |v_i^{SSF} - v_j^{SSF}| \leq 2\}$$

- $v_i^{SSF}$：個體 $i$ 的信仰價值
- $v_j^{SSF}$：個體 $j$ 的信仰價值
- $N(i)$：個體 $i$ 的所有鄰居
- $R$：被推薦的鄰居集合

### 4. 雙重記憶系統

因為需要處理大量的個體互動，每個個體具備：

| 記憶類型 | 符號 | 內容 |
|---|---|---|
| 長期記憶 | $m_i^l$ | 儲存過去互動歷史的摘要 |
| 短期記憶 | $m_i^s$ | 儲存當天對話內容 |

### 5. 意見更新

在第 $t$ 天結束時，每個個體根據：
- **自身理念**
- **與他人交流**
- **雙重記憶系統的推理反思**

透過 **GPT-4o-mini（OpenAI API）** 進行推理，更新自身對於該主題的想法，並產生：

$$v_i \in [-2, 2]$$

- $-2$：完全反對
- $+2$：完全贊同

## 三、對我們的限制

論文1的實作方式，對我們來說有以下限制：

#### 一、成本問題
直接 call OpenAI API 意味著每次模擬都需要付費，若要大規模模擬，API 呼叫次數龐大，成本難以控制。

#### 二、可重現性問題
OpenAI 模型會不定期更新，同樣的 Prompt 在不同時間可能產生不同結果，導致實驗結果難以重現，不符合學術研究的嚴謹性。

#### 三、數值來源不透明
論文1未說明 GPT-4o-mini 的 Prompt 格式，以及 LLM output 如何轉換為可用數值（$v_i \in [-2, 2]$），導致數值來源無法驗證。

#### 四、缺乏 Rewiring 機制
論文1的網路架構**固定**，只透過推薦演算法決定互動對象，未考慮個體主動調整連結的行為，與真實世界不符。

## 四、與論文2的連結

論文2正是針對上述限制提出改進：

| 限制 | 論文1 | 論文2 的改進 |
|---|---|---|
| 網路結構 | 固定，無 Rewiring | 動態調整，加入 Follow / Unfollow |
| 數值透明度 | 直接 call API，Prompt 格式未說明 | 定義 $T_o$、$T_r$、$T_g$ 三種 Prompt Template 架構 |
| 驗證方式 | 僅與 BCM / FJ 比較 | 加入真實 Twitter 資料對照 |
| LLM 比較 | 只用 GPT-4o-mini | 比較 6 種 LLM |
